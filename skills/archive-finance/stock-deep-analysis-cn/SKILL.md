# A股个股全面分析总控 Skill

这是一个**总控型 skill**，不重复造轮子，而是显式编排 5 个现有 skills：

1. `akshare-open` —— 综合数据与增强分析底座
2. `akshare-stock-data` —— 基础数据查询与接口兜底
3. `stock-analysis-cn` —— 单票技术面/财务面分析模板
4. `finance-news-cn` —— 新闻检索、摘要、催化核验
5. `stock-monitor-anomaly` —— 异动、量能、换手、波动筛查

核心原则：
- **先数据，后结论**
- **先事实，后解释**
- **先多维交叉验证，后下判断**
- 某一模块缺数据时，不停止整份分析；明确标注缺口，并继续完成剩余部分
- 默认优先适配**短线主板交易模式**，不是长篇泛研究报告模式

## 默认模式切换

### 模式 A：短线主板交易型（默认优先）
当用户关注：
- 这票明天能不能看
- 是否适合纳入候选池
- 当前是否属于低位趋势成形
- 板块、催化、承接是否匹配
- 支撑/压力、盈亏比、操作策略

则优先输出“交易型深度分析”，重点回答：
- 所属板块与板块阶段
- 个股角色（龙头/中军/补涨/跟风）
- 趋势结构是否成立
- 催化是否可信
- 承接与量价是否匹配
- 次日计划买点 / 确认条件 / 放弃条件 / 失效条件

### 模式 B：研究型深度分析
当用户明确要求：
- 做完整基本面研究
- 全面分析公司长期价值
- 财务、估值、事件、宏观都要细讲

再展开更完整的研究型长报告。

## 何时使用

当用户想要对单只股票做完整研判时使用，例如：
- “帮我全面分析康强电子”
- “这只票值不值得关注”
- “从基本面、技术面、资金面、新闻面都看一下”
- “做一份完整个股分析”
- “帮我看看这只股票现在是什么状态” 

## 输入要求

优先拿到：
- 股票代码（如 `002119`）
- 或股票名称（需先查代码）

如用户只给公司名，先查代码再继续。

## 标的范围约束

默认按用户当前体系做以下约束：
- 主板股票优先：`600/601/603/605/000/001/002/003`
- 默认排除：`300/301`（创业板）、`688/689`（科创板）、`8/4` 开头（北交所等）
- ETF/LOF 允许分析，但必须走“场内基金产品分支”，不要与普通公司股票共用基本面框架

如果用户明确要求分析非主板个股，再解除过滤；否则应先提示“当前默认主板框架”。

## 分析总流程（强制顺序）

### 0. 识别标的
- 若用户给的是名称：先查证券代码
- 若用户给的是代码：确认名称、行业、市场
- **代码查询兜底**：若 `ak.stock_info_a_code_name()` 因上游连接/TLS/reset 失败，不要卡住；改用东方财富搜索建议接口兜底：
  ```bash
  python - <<'PY'
  import requests, urllib.parse
  name='康强电子'
  url=f'https://searchapi.eastmoney.com/api/suggest/get?input={urllib.parse.quote(name)}&type=14&count=10'
  print(requests.get(url, timeout=20).text)
  PY
  ```
  从返回的 `QuotationCodeTable.Data` 中提取 `Code`、`Name`、`SecurityTypeName` 等字段。

### 1. 基础信息与原始数据底座
优先调用：`akshare-open` + `akshare-stock-data`

至少获取：
- 股票名称 / 代码 / 行业 / 上市时间
- 最新价、总市值、流通市值
- 历史行情（至少近 6-12 个月）
- 财务指标与财务摘要
- 板块/行业归属

### 2. 基本面分析
优先调用：`akshare-open` 的 `fundamental_analysis.py`，并参考 `stock-analysis-cn`

必须覆盖：
- 营收、利润、EPS
- ROE、毛利率、资产负债率
- 经营现金流 / 每股经营现金流
- 若是银行/券商/保险等金融股，要按行业特性解读，不机械套用普通工业股逻辑

输出：
- 基本面亮点
- 基本面短板
- 盈利质量判断

### 3. 技术面 / 趋势面
优先调用：`stock-analysis-cn` + `akshare-open` 的 `strategy_analysis.py`

必须覆盖：
- 近 1/3/6/12 个月涨跌与波动
- MA5 / MA20 / MA60 或等价趋势判断
- 短中期趋势：多头 / 空头 / 震荡
- 是否放量上行、缩量整理、放量下跌
- 若历史行情接口异常，要明确写“趋势模块部分缺失”

### 4. 新闻面 / 催化面
优先调用：`finance-news-cn` + `akshare-open` 的 `news_summary.py`

必须覆盖：
- 最近 3-10 条与个股直接相关的重要新闻
- 是否有业绩、订单、产品、政策、行业、资金、调研、回购等催化
- 传闻与正式来源分开写
- 未找到直接催化时要明确写出

### 5. 事件面
优先调用：`akshare-open` 的 `event_driven_analysis.py`

必须覆盖：
- 公告
- 龙虎榜
- 机构调研
- 股东变化
- 回购
- 研报

输出：
- 已知事件清单
- 可能的正向催化
- 可能的风险事件

### 6. 资金面
优先调用：`akshare-open` 的资金流脚本

至少看：
- 个股资金流
- 主力资金流
- 行业/概念板块资金流
- 北向 / 沪深港通相关（如果适用）

输出：
- 资金是流入、流出，还是分歧
- 资金面是否支持当前走势和题材逻辑

### 7. 量能异动 / 微观行为
优先调用：`stock-monitor-anomaly`

至少看：
- 涨跌幅
- 换手率
- 量比
- 振幅
- 是否属于异动股

输出：
- 是否存在短线异常活跃
- 是趋势确认，还是情绪冲高

### 8. 板块面 / 情绪面 / 大盘面
优先调用：`akshare-open`

必须覆盖：
- 所属行业 / 概念板块近期强弱
- 板块是否处于资金关注主线
- 大盘环境（指数 / 市场总貌）是否支持个股表现
- 市场情绪是否偏强、偏弱、分化

### 9. 宏观面
按需调用：`akshare-open` 宏观数据 + `finance-news-cn`

规则：
- 如果个股受宏观利率、地产、周期、出口、汇率、消费、政策影响明显，则必须补宏观
- 如果是小票题材股且宏观影响弱，可以简写

### 10. 估值面
优先调用：`akshare-open` 的 `valuation_analysis.py`

规则：
- 如果估值接口有值，就给出 PE/PB/PS 的大致判断
- 如果估值接口无值或字段失效，要明确写“估值模块数据不足”
- 不要因为估值缺失而中断整份分析

## 输出结构（固定）

最终必须按以下结构输出：

### 默认：短线主板交易型输出
1. **一句话结论**
2. **核心观点摘要**（3-8 条）
3. **所属板块 / 板块阶段 / 个股角色**
4. **技术面 / 趋势面 / 量价承接**
5. **新闻与催化**
6. **事件面**
7. **资金面**
8. **量能与异动**
9. **板块 / 情绪 / 大盘环境**
10. **关键支撑位 / 压力位 / 盈亏比**
11. **次日计划：买点 / 确认条件 / 放弃条件 / 失效条件**
12. **综合判断**
13. **还需要补什么数据**（如有）

### 若用户明确要求研究型深度报告，再补充
- 基本面
- 宏观影响
- 估值与风险

## 结论模板要求

最后结论至少回答：
- 这只股票当前更像：价值型 / 趋势型 / 题材型 / 事件驱动型 / 高波动博弈型？
- 当前驱动它的核心矛盾是什么？
- 现在最值得继续跟踪的是哪 3 个变量？

## 数据缺失处理规则

- 若 `akshare-open` 子模块失败：继续用其他 4 个 skills 和原始接口补位
- 若历史行情失败：保留基本面、新闻、事件、资金、板块分析
- 若估值失败：明确写“估值数据不足”
- 若新闻不足：明确写“未找到足够直接催化”
- 若事件不足：明确写“暂无显著公开事件信号”

## 不允许的行为

- 不能只看一个模块就下结论
- 不能把新闻热度等同于投资价值
- 不能把异动等同于买点
- 不能把分析写成确定性荐股承诺

## 推荐执行顺序（简版命令思路）

1. `akshare-open`：拿综合底层数据和增强模块
2. `akshare-stock-data`：补基础行情/财务接口
3. `stock-analysis-cn`：补技术面与财务解释
4. `finance-news-cn`：补新闻摘要和催化核验
5. `stock-monitor-anomaly`：补量价异动和短线活跃度
6. 汇总输出最终分析

## 适用定位

这个 skill 适合做：
- 个股深度初筛
- 盘后复盘
- 研究初稿
- 多维信息整合后的观察结论

不适合做：
- 自动下单
- 个性化投资建议
- 忽略数据缺口的绝对性结论
